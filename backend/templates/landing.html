WHAT YOU CAN DO

Every feature, run by the same model

Twelve tools, one shared feature-engineering pipeline underneath. Nothing here is a static mockup — each tab calls a live prediction endpoint.

🎯

Single Predictor

Paste a 20bp guide RNA and a target window. Get a risk score, a LOW / MEDIUM / HIGH classification, and a base-by-base mismatch trace showing exactly where the guide and target disagree.

/api/predict
🧬

Bulk Comparator

Drop in a list of candidate target sequences and rank them all against one reference guide in a single pass, sorted from safest to riskiest.

/api/compare
🌐

Genome Track Viewer

Plot a synthetic off-target risk density curve across chromosome coordinates, so you can see how risk theoretically shifts across a genomic region.

/api/genome-track
🧠

Attention-Style Matrix

A position-weighted heatmap showing how much each base pairing contributes to the model's decision — inspired by attention maps from sequence models.

/api/attention-matrix
📈

Historical Analytics

A rolling 15-day chart of average predicted risk across demo runs, for tracking how scoring behaves over a session or a class demo.

/api/historical-analytics
💬

Genomics Q&A Assistant

Ask a plain-English question about gene editing and get a 5-point explanation back, powered by an LLM (with an offline fallback when no API key is set).

/api/chat
🛡️

SafeCRISPR Score

A single composite 0–100 safety rating built from the model's risk probability, PAM validity, seed-region mismatches, and GC balance — one number, easy to compare across guides.

/api/safecrispr-score
📋

Risk Factor Breakdown

Turns the Random Forest's feature importances into plain-English sentences explaining exactly why a guide scored the way it did — mismatch count, seed-region position, PAM validity, and GC content.

/api/risk-factors
💡

Alternative Guide Recommender

Tests single-base corrections against the live model and surfaces whichever variant carries the lowest predicted risk — including cases where "fixing" a mismatch inside the PAM actually backfires.

/api/recommend-alternative
📊

SHAP-Style Explanation

A dependency-free, marginal-contribution feature attribution plot: each feature is swapped from a dataset baseline to its real value to measure how much it shifts the model's predicted risk.

/api/shap-explain
⚖️

Multi-Model Comparison

Runs the same guide RNA through three classifiers trained on identical features — Random Forest, Logistic Regression, and a small Neural Net — and shows where they agree or disagree.

/api/compare-models
🧪

Model Performance & Failure Analysis

Holds out 30% of the model's own training data, fits a fresh classifier on the rest, and reports a confusion matrix, precision/recall/F1, an ROC curve, and the held-out cases it got most confidently wrong.

/api/model-performance
UNDER THE HOOD

What the model actually does

No black box. Here's the real pipeline, in order, every time you hit "Run."

1

Align & encode

Your guide RNA and target sequence are aligned base by base. Each position gets a match/mismatch flag and a position weight that rises toward the cut site.

2

Extract 3 features

Total mismatch count, mismatches inside the 8bp seed region nearest the PAM, and overall GC content of the guide.

3

Random Forest scores it

A 100-tree Random Forest classifier — trained on real lab data when a CSV is supplied, otherwise on baseline rules — outputs a risk probability.

4

PAM sanity check

The model checks the target's final 3 bases for a valid SpCas9 "NGG" PAM motif. No valid PAM sharply cuts the risk score, since cutting can't physically occur.

Heads up: this is a learning tool, not a lab-validated predictor. Without a real crispr_experimental_data.csv of measured cleavage efficiencies, the Random Forest trains on simple synthetic rules (mismatch count + seed-region logic), not actual biology. Treat every score here as a demonstration of how ML-based off-target prediction works — not as guidance for an actual edit.
